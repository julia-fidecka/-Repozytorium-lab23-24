Czym jest Git?

Git to system kontroli wersji, który pozwala na śledzenie zmian w plikach w czasie. Git umożliwia współpracę wielu osób nad jednym projektem, zarządzanie wersjami plików, a także odzyskiwanie poprzednich wersji kodu.

Historia Git

Git został stworzony w 2005 roku przez Linusa Torvaldsa, twórcę jądra Linux. Powstał jako odpowiedź na potrzebę lepszego systemu kontroli wersji dla rozwoju jądra Linux.

Podstawowe komendy Git

Git oferuje szereg komend, które pomagają zarządzać repozytoriami i wersjami kodu. Oto kilka z nich:

  • git init - Inicjalizuje nowe repozytorium Git w bieżącym katalogu.
  • git add plik - Dodaje zmiany w pliku do obszaru staging (czyli do przygotowania do commitowania).
  • git commit -m "wiadomość" - Tworzy nowy commit, zapisując zmiany w repozytorium.
  • git status - Sprawdza status repozytorium, pokazując zmiany w plikach.
  • git log - Wyświetla historię commitów w repozytorium.

Zalety korzystania z Git

  • Śledzenie historii zmian – łatwe przeglądanie wszystkich modyfikacji w projekcie.
  • Rozproszona architektura – każdy użytkownik ma lokalną kopię repozytorium.
  • Współpraca zespołowa – łatwe zarządzanie zmianami od wielu użytkowników.
  • Obsługa gałęzi (branching) – możliwość równoległego rozwijania różnych funkcjonalności.
  • Ochrona przed utratą danych – bezpieczne cofanie zmian i odzyskiwanie stanu projektu.
  • Bezpłatne i szeroko wspierane narzędzie z dużą społecznością.

GitHub - platforma współpracy

GitHub to popularna platforma do hostingu repozytoriów Git, która ułatwia współpracę, zarządzanie wersjami kodu, oraz integrację z innymi narzędziami i usługami. GitHub umożliwia łatwe tworzenie pull requestów, zarządzanie problemami (issues) i wiki projektów.
